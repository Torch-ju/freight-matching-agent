# 满帮动态决策智能体项目一页纸总结

## 一句话说明

这是一个面向货运连续找货场景的 Agentic AI 在线决策系统：Agent 在每一步只能通过官方接口观察司机状态、候选货源、历史动作和自然语言偏好，然后选择 `take_order`、`wait` 或 `reposition`，目标是在完整月度周期内提高净收益并降低个性化偏好罚分。

## 项目为什么做

真实货运场景中，司机找货不是单次推荐问题，而是连续经营问题。每一次接单都会改变司机的位置、时间、疲劳/休息状态和后续可见货源机会。同时，司机偏好往往以自然语言表达，例如休息时段、长途上限、指定货源类型目标、跨月补偿任务、禁忌品类和地点任务等。

因此，本项目要解决的是：

- 如何把自然语言偏好转成程序可执行约束。
- 如何在收益、成本、时间窗和偏好风险之间做统一决策。
- 如何在官方 online-only 约束下，不读取全量原始数据，只基于运行时接口完成连续动作选择。

## 系统怎么做

最终系统分为五层：

```text
官方 SimulationApiPort
  -> 司机状态 / 候选货源 / 历史动作 / LLM 调用
  -> HybridPolicy 策略编排
  -> Preference Compiler 偏好编译器
  -> Driver Progress Ledger 司机进度账本
  -> ServiceNode / ServicePlan 短视界规划
  -> take_order / wait / reposition
```

核心模块：

- **偏好编译器：** 将自然语言偏好归纳为时间窗、地点任务、数量目标、跨月补偿、数量上限、禁忌规则等通用偏好类型，再转成运行时可执行规则。
- **司机进度账本：** 记录某类偏好任务已经完成多少、还差多少、是否触发上限或风险，用于后续评分和规划。
- **ServicePlan：** 将真实货源、等待、空驶和偏好虚拟任务统一成服务节点，在有限视界内综合比较收益、成本、时间窗和偏好目标。
- **参数实验 runner：** 自动运行本地参数组合，输出毛利、成本、净利、偏好罚分、Token、动作日志和最优试验记录。
- **工程边界：** 决策代码不读取 `cargo_dataset.jsonl`、`drivers.json` 或官方 server 内部数据，不使用离线全量市场先验影响动作。

## 做出了什么结果

已知关键结果：

| 来源 | 指标 |
| --- | --- |
| 本地公开 demo 最高净利 | `114856.46` |
| 本地公开 demo 对应偏好罚分 | `8000.0` |
| 参数实验 runner 本地最好 trial | `108923.34` 净利，`3000.0` 偏好罚分 |
| 平台隐藏集已知最好 | `25108.30` |
| 失败高罚分平台结果 | `208300.0` 偏好罚分，且两次结果完全一致 |

这些结果说明：本地公开 demo 上系统已经具备较强收益能力，但平台隐藏集暴露出自然语言偏好编译泛化不足的问题。最终没有获胜，核心瓶颈不是单纯货源收益判断，而是隐藏偏好没有稳定转成可执行约束。

## 最重要的技术结论

1. **Agent 决策不能只做单票排序。**  
   货运找货是序贯优化问题，每个动作会影响后续时间、位置和机会，需要短视界规划。

2. **LLM 不能直接决定动作。**  
   LLM 更适合低频语义抽取；最终约束、账本、评分函数和动作必须由确定性程序控制。

3. **自然语言偏好编译是核心难点。**  
   如果偏好解析错误，越强的运筹优化越可能放大错误，导致平台高罚分。

4. **本地高分不等于隐藏集泛化。**  
   本地公开偏好覆盖有限，必须用 paraphrase、组合偏好、反事实偏好和 adversarial preference 构造泛化测试。

## 可复用资产

- `src/preferences/`：自然语言偏好到规则 DSL / ledger / runtime 的原型。
- `src/planning/service_node.py`、`src/planning/service_plan_optimizer.py`：ServiceNode / ServicePlan 短视界规划框架。
- `scripts/run_parameter_experiments.py`：参数实验 runner。
- `tests/`：偏好解析、规则运行、规划器和策略契约测试。
- `docs/final-review/`：比赛复盘、技术路线总结、数据清单、清理报告和简历表述。

## 可继续研究的问题

本项目赛后沉淀出的可研究问题是：

> 隐藏偏好自然语言如何稳定编译成可执行约束，并与在线运筹决策无缝连接？

后续可以围绕自然语言偏好编译 benchmark、Preference IR 到 executable DSL 的可验证转换、多 LLM consensus、偏好泛化测试集和在线约束优化继续推进。
